# 页面交互实现要点

## 通用
- 面板 Hover：背景从 `surface` 到 `surfaceHover`，180ms。
- 选中态：左侧 2px 品牌绿指示线 + 轻微绿色内阴影，不使用大面积发光。
- 数字统一使用 tabular numbers，行情变化保持绿涨红跌。
- Skeleton 应保持面板结构，不使用全屏闪烁。

## 首页 / 工作区概览
- 待审核、运行中、告警、完成数字卡可点击过滤任务列表。
- Watchlist 行 Hover 显示打开详情与创建分析操作。
- 最近活动保持时间线结构，增量更新时只淡入新行。

## 市场分析工作台
- 执行进度为真实步骤状态机：完成、进行、等待、失败、阻断。
- 证据门禁和风险门禁分别展示，不合并为一个“AI 分数”。
- 人工复核按钮必须明确区分：通过并发布、驳回、编辑。
- 请求提交、重试和人工复核使用幂等键；重复点击时按钮进入锁定状态。

## 运行记录
- 左侧筛选器支持保存条件。
- 点击列表行打开右侧详情抽屉，不丢失当前筛选与滚动位置。
- 状态颜色只用于标签和关键数值，避免整行强色背景。

## 审核收件箱
- 默认按即将过期优先，其次按风险等级。
- 打开一项后右侧固定详情；通过或驳回后自动切换下一项。
- 过期倒计时使用橙色，低于 5 分钟切换危险色但不闪烁。

## 报告资料库 / 报告详情
- 表格和卡片视图共享同一过滤条件。
- 报告详情的版本历史始终可见，Fork、重跑和导出属于次级操作。
- Evidence、Risk、Web Sources、Data Lineage 应支持锚点跳转。

## 设置
- 保存后显示右上角 Toast；失败时保留用户输入。
- Bark key / Webhook secret 默认掩码，不在页面重新回显完整值。
- 危险操作与普通设置分区，删除类操作使用二次确认。
