# 测试与验收边界

## 测试目标

确认 v1 满足：

- 固定加载 skill。
- 每轮刷新事实。
- 输出严格 JSON。
- 风控 fail closed。
- journal 完整。
- 通知失败不影响 verdict。
- 历史记忆不污染新判断。
- Leader 规划、并发 researcher、Leader 四角色审查真实进入 prompt 和 journal。

## 单元测试

### RequestParser

覆盖：

- 手动 query 解析。
- 定时 query 解析。
- 缺 symbol。
- 缺 horizon。
- 不支持 symbol。

### SkillRuntime

覆盖：

- skill path 存在。
- `SKILL.md` 缺失。
- reference 缺失。
- skill hash 计算。
- `okx_snapshot.py` 缺失。
- script 超时。

### EvidenceCollector

覆盖：

- OKX snapshot 正常。
- OKX snapshot 返回部分字段。
- OKX snapshot 超时。
- 核心事实缺失进入 hard block。
- web fallback disabled 被明确记录。
- 核心事实 stale 进入 research fallback。
- candles stale 使用 `candle_bar + stale_market_data_seconds` 阈值。

### Research / Leader 编排

覆盖：

- `ResearchPlanner(static)` 生成固定查询。
- `ResearchPlanner(llm)` 调用 OpenAI-compatible `/v1/chat/completions` 并解析 strict JSON。
- LLM planner 失败时回落 static，并记录 fallback 原因。
- `execute_research` 按 `max_workers` 并发执行多个 query。
- `responses_web_search` 调用 `/v1/responses` 且 `tool_usage.web_search.num_requests > 0`。
- `LeaderResearchSynthesizer(llm)` 接收并发搜索结果，输出 `leader_finalizer`、`bull_reviewer`、`bear_reviewer`、`data_quality_reviewer`、`execution_risk_reviewer`。
- LLM leader 失败时回落 static，并记录 fallback 原因。

### StrictPlanParser

覆盖：

- 合法 JSON。
- fenced JSON 拒绝或明确按配置处理。
- 多余文本拒绝。
- 非法 enum。
- 缺 required fields。
- 数值字段类型错误。
- `manual_execution_required=false`。

### RiskGate

覆盖：

- 无 stop 的 open / trigger / flip。
- open / trigger / flip 缺 mark、index、order book。
- probability 超过 cap。
- max leverage 超限。
- risk pct 超限。
- plan 过期。
- stale data。
- forbidden env 存在。

### Memory

覆盖：

- episode 切换。
- session summary 不保存市场事实。
- journal 不回灌。
- active event TTL 过期。

### Notification

覆盖：

- Bark 成功。
- Bark 失败。
- 内容截断。
- 敏感信息脱敏。
- blocked 计划通知文案。

### Scheduler

覆盖：

- 拿锁成功。
- 拿锁失败跳过。
- job 异常不打断循环。
- max iterations。

## 集成测试

### Fixture 完整链路

```text
fixture market data
fixture decision output
strict parser
risk gate
journal
noop notifier
```

验收：

- journal 有完整 payload。
- verdict 正确。

### OpenAI-Compatible 链路

```text
fixture evidence
real compatible model
strict parser
risk gate
noop notifier
```

验收：

- 模型输出能被解析。
- 失败时 block。

### Research Fallback 链路

```text
missing/stale market data
LLM/static research planner
concurrent search adapter
evidence synthesizer
LLM/static leader reviewer
final decision model
risk gate
journal
```

验收：

- search-derived 结果进入 `market_snapshot.points.web_*`。
- `research.plan`、`research.results`、`research.leader_summary` 写入 journal。
- 缺核心执行事实时仍由 RiskGate 阻断开仓类动作。

### Bark 链路

```text
fixture plan
real Bark key
send notification
record result
```

验收：

- 手机收到提醒。
- journal 记录 notification result。

## 本地验收

本地开发完成后必须跑：

```powershell
pytest -q
crypto-alert show-config
crypto-alert run-once --symbol ETH-USDT-SWAP
crypto-alert test-bark
```

## Docker 验收

必须验证：

```powershell
docker compose config
docker compose up -d
docker compose logs
```

要求：

- 不暴露 host port。
- 不固定 container name。
- data 目录持久化。
- 环境变量注入。

## 服务器验收

海外服务器部署前必须确认：

- OKX public API 可达。
- LLM base URL 可达。
- Bark / 飞书可达。
- Docker Compose 不和已有服务冲突。
- journal 可写。

## 验收通过标准

- 所有测试通过。
- 本地 fixture 跑通。
- 本地 LLM 跑通。
- Bark 跑通。
- Docker Compose 跑通。
- OKX 不可达时能 fail closed。
- JSON 非法时能 fail closed。
- journal 写入失败时不假装成功。
