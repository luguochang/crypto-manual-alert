# 正式文档索引

## 目的

`docs/formal/` 是当前开发、验收和后续维护的正式文档目录。

`docs/` 根目录和早期讨论文档只作为历史参考；开发实现、测试边界、部署说明和架构决策以本目录为准。

## 当前最终决策

当前文档权威关系如下：

- `22-完整Agent业务流程与自进化评估架构设计.md`：业务流程总纲，说明手动 query、定时 query、trace/eval、人工发布闭环等业务范围。
- `29-Agent与Skill拆分详细设计.md`：受控 Agent Swarm 的目标架构和完成定义，是 Agent/Skill/Policy/Gate 设计的最高优先级文档。
- `30-受控AgentSwarm-MVP实施契约.md`：当前代码事实、兼容路径和边界契约参考，约束不能宣称什么、不能绕过什么。
- `31-受控AgentSwarm主链收敛与质量切换计划.md`：历史执行进度入口，记录已完成的 shadow audit、candidate artifact、release gate 和前端基础可视化改造。
- `32-架构功能收敛总计划与追踪清单.md`：历史重构总计划说明，保留模块归属、质量清单和最终完成定义作为背景参考。
- `34-生产级AgentSwarm优化目标与执行计划.md`：当前新的执行入口，后续开发按该文档的检查点推进。
- `35-剩余主缺口对抗审查与执行清单.md`：当前剩余主缺口的多角色对抗审查、阶段计划、checkpoint checklist 和暂停条件。注意其 §2 的"多角色对抗审查"是单模型角色扮演，非真实独立 subagent。
- `36-成熟观测与评测平台接入方案.md`：Langfuse + DeepEval 混合观测架构提案。Phase B（Run Detail 收缩为 Cockpit）纳入交付；Phase D/E（外部平台接入）推迟到 v1 交付后。
- `37-真实多Agent对抗审查与交付方向裁决.md`：项目第一次真实多 Agent 对抗审查（5 路审查 + 逐条对抗验证）的结论与方向裁决。裁决 partly-off-track；记录确认的核心问题、被反驳的误判、交付关键路径、对 doc 35/36 的处置和冻结项。后续执行以本文裁决为准，直到其 §7 交付闭环跑通。

涉及当前代码事实、兼容路径和边界契约时，以 `30-受控AgentSwarm-MVP实施契约.md` 的“当前态索引与历史记录解释规则”为参考；旧执行记录、旧测试路径和旧 wrapper 描述只作为历史参考。

涉及后续重构执行顺序、进度追踪、质量切换和汇报节奏时，以 `34-生产级AgentSwarm优化目标与执行计划.md`、`35-剩余主缺口对抗审查与执行清单.md` 和 `37-真实多Agent对抗审查与交付方向裁决.md` 为准；`37` 的方向裁决优先级高于 `35` 的阶段计划。不再继续把流水记录堆入 29/30/31/32。

当前最终方案是：

```text
确定性业务状态机
  + 受控 Agent Swarm 目标架构
  + 固定 crypto-macro-decision 方法论拆分后的 Skill/Policy/Gate
  + LeadAgent 明确分配任务
  + 独立 Worker Agent 并发贡献 EvidencePacket / AgentContribution
  + 强制 Parser / Facts / PlanSemantic / Risk gates
  + 全链路 trace / frozen input / eval
  + 人工审批的 prompt / rule / workflow 改进闭环
```

当前代码仍不是受控 Agent Swarm。当前真实形态是：

```text
RunExecutor
  -> LegacyPlanRunnerAdapter
  -> PlanRunner / LegacyDecisionWorkflow
  -> fixed legacy pipeline
  -> concurrent search
  -> single leader review for legacy research fallback
  -> workflow.pre_final_orchestration
  -> orchestration.shadow_audit(decision_effect=none, before final decision)
  -> final decision
  -> parser / production_control_gate / risk
  -> journal / notification
```

现有兼容链路仍作为兼容入口和迁移基线，但 `PlanRunner` 后续必须降级为 legacy adapter，不能继续作为长期主编排入口。

工程落地以 `23-TypeScript与Python落地技术栈方案.md` 为技术栈约束：

```text
Next.js TypeScript UI
  -> Python FastAPI
  -> SQLite 默认 / PostgreSQL 可选
  -> Python Agent Workflow
  -> Bark
```

第一轮落地不引入额外中间件。

实现质量以 `24-生产级实现计划与公开仓库规范.md` 为代码和公开仓库约束。

当前执行阶段以 `34-生产级AgentSwarm优化目标与执行计划.md` 的当前进度快照、Checkpoint checklist 和验收命令为准；`30-受控AgentSwarm-MVP实施契约.md` 的当前态索引和结构测试作为代码事实与边界契约参考。当前生产最终输入仍是 `legacy_prompt`，Agent Swarm/DecisionInput/candidate final 仍是候选、审计和评测路径：

```text
DecisionRequest
  -> RunExecutor creates DecisionRunContext
  -> SideEffectPolicy from run_type
  -> LegacyPlanRunnerAdapter
  -> PlanRunner / LegacyDecisionWorkflow
  -> pre-final orchestration shadow audit
  -> existing API / Journal / Trace contract
```

## 不再作为当前实现依据的方向

- 完整 7-agent swarm 平台。
- 动态发现任意 skill。
- 任意脚本自动注册为工具。
- 长期向量记忆。
- journal 默认回灌到 live decision。
- 自动交易。
- 把并发 search query 或单次 leader JSON reviewer key 宣称为已完成 agent swarm。
