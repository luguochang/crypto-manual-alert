# 正式文档索引

## 目的

`docs/formal/` 是当前开发、验收和后续维护的正式文档目录。

`docs/` 根目录和早期讨论文档只作为历史参考；开发实现、测试边界、部署说明和架构决策以本目录为准。

## 当前最终决策

当前以 `22-完整Agent业务流程与自进化评估架构设计.md` 为下一阶段重构总纲。核心方案是：

```text
确定性业务状态机
  + 固定 crypto-macro-decision skill
  + 可配置 Agent 研究/审查层
  + 强制 Parser / Facts / PlanSemantic / Risk gates
  + 全链路 trace / frozen input / eval
  + 人工审批的 prompt / rule / workflow 改进闭环
```

旧 v1 链路仍作为兼容入口和迁移基线：

工程落地以 `23-TypeScript与Python落地技术栈方案.md` 为技术栈约束：

```text
Next.js TypeScript UI
  -> Python FastAPI
  -> SQLite 默认 / PostgreSQL 可选
  -> Python Agent Workflow
  -> Bark
```

首版不引入额外中间件。

实现质量以 `24-生产级实现计划与公开仓库规范.md` 为代码和公开仓库约束。

```text
Query / Scheduler
  -> Parse
  -> EpisodeCheck
  -> Load crypto-macro-decision Skill
  -> RefreshFacts via skill tools
  -> CoreFactsGate
  -> if missing/stale: LeaderPlanResearch
  -> ConcurrentResearchers via configured SearchAdapter
  -> LeaderEvidenceSummary + 4-role AdversarialReview
  -> FinalDecision
  -> StrictParser
  -> RiskGate
  -> Journal
  -> Bark / Feishu
```

## 不再作为 v1 依据的方向

- 完整 7-agent swarm 平台。
- 动态发现任意 skill。
- 任意脚本自动注册为工具。
- 长期向量记忆。
- journal 默认回灌到 live decision。
- 自动交易。
