# 记忆与多轮对话

## 目标

支持多轮对话，同时避免旧结论污染新判断。

## v1 记忆层

v1 只保留：

- `SessionContext`
- `ActiveEventPool`
- 静态 `LessonRules`
- `AuditJournal`

## SessionContext

只保存：

- session id
- episode id
- symbol
- horizon
- position_state
- latest user intent
- very short summary

不保存：

- 旧价格
- 旧 funding
- 旧 OI
- 旧 liquidation
- 旧方向结论作为事实

## ActiveEventPool

必须包含：

- event id
- symbol / asset scope
- status
- last_seen_at
- ttl
- source

过期事件必须自动失效。

## LessonRules

v1 使用静态规则，不自动写入长期 lesson。

允许：

- “缺少 mark/index 时压低置信度”
- “不要复用 stale OI”
- “reviewer 必须处理反向链”

禁止：

- “昨天看多所以今天看多”
- “某次交易成功所以同模式继续”

## AuditJournal

只追加，不默认回灌。

用途：

- 审计
- 复盘
- 调试

禁止：

- 默认从 journal 检索旧结论作为 live decision 输入。

## Episode 切换

必须新建 episode：

- symbol 变化
- horizon 明显变化
- 持仓方向或状态变化
- 合约/交易所变化
- 任务类型变化
- 重大事件窗口变化
- 新一轮独立判断

可以继续 episode：

- 同一 symbol、同一 horizon、同一持仓下追问细节。
- 用户要求复核当前计划。

## 多轮状态机

```text
Parse -> EpisodeCheck -> RefreshFacts -> Deliberate -> Guard -> JournalNotify
```

允许回跳：

- facts stale -> RefreshFacts
- episode changed -> EpisodeCheck

不允许：

- journal -> Deliberate
- old conclusion -> FinalDecision
- stale summary -> RiskGate

## 防污染规则

1. 旧市场事实不能作为 live fact。
2. 旧方向结论不能作为新一轮证据。
3. session summary 只能帮助理解用户意图。
4. active events 必须带 TTL。
5. audit journal 默认只写不读。
