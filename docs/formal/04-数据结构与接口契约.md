# 数据结构与接口契约

## 目标

定义 v1 的核心数据对象，避免模型输出、风控、通知和 journal 各自使用不同结构。

## DecisionRequest

入口标准化后的请求对象。

```json
{
  "request_id": "uuid",
  "source": "manual|scheduled",
  "session_id": "uuid",
  "symbol": "ETH-USDT-SWAP",
  "horizon": "6h",
  "position_state": "long|short|none|unknown",
  "raw_query": "用户原始输入或定时任务 prompt",
  "created_at": "2026-06-26T00:00:00Z"
}
```

规则：

- `symbol` 必须在 allowed symbols 内。
- `horizon` 必须明确。
- `position_state=unknown` 时只能生成 `no trade` 或要求补充，不能直接生成开仓计划。

## SessionContext

当前会话和 episode 的最小上下文。

```json
{
  "session_id": "uuid",
  "episode_id": "uuid",
  "symbol": "ETH-USDT-SWAP",
  "horizon": "6h",
  "position_state": "long",
  "latest_user_intent": "manual_operation_plan",
  "latest_summary": "用户当前持有 ETH 多单，希望看 6 小时内的操作方案"
}
```

禁止保存：

- 旧价格
- 旧 funding
- 旧 OI
- 旧清算数据
- 旧方向结论作为当前事实

## SkillContext

固定 skill 的加载结果。

```json
{
  "skill_name": "crypto-macro-decision",
  "skill_path": "third_party/skills/crypto-macro-decision",
  "skill_hash": "sha256",
  "loaded_references": [
    "data-sources.md",
    "exchange-derivatives.md",
    "templates.md"
  ]
}
```

## EvidenceSnapshot

v1 的事实和数据质量对象。

```json
{
  "request_id": "uuid",
  "symbol": "ETH-USDT-SWAP",
  "fetched_at": "2026-06-26T00:00:00Z",
  "facts": {
    "last": 3500.0,
    "mark": 3499.0,
    "index": 3498.0,
    "funding_rate": 0.0001,
    "open_interest": 100000.0,
    "candles_1h": [],
    "candles_4h": [],
    "order_book": {}
  },
  "sources": [
    {
      "name": "okx_snapshot",
      "type": "exchange-native",
      "timestamp": "2026-06-26T00:00:00Z"
    }
  ],
  "unavailable": [],
  "stale": [],
  "hard_blocks": [],
  "soft_downgrades": [],
  "confidence_cap": 0.58
}
```

规则：

- 所有事实必须有来源。
- 核心事实缺失必须进入 `hard_blocks` 或 `confidence_cap`。
- 不允许把 web-derived 数据标成 exchange-native。

## RootCauseReview

根因链和反链对象。

```json
{
  "primary_chain": {
    "observed_fact": "fresh fact only",
    "mechanism": "why this fact can move price",
    "predicted_path": "most likely path if mechanism continues",
    "confirmation": "what confirms this path",
    "invalidation": "what invalidates this path"
  },
  "counter_chain": {
    "observed_fact": "fresh fact only",
    "mechanism": "opposite explanation",
    "predicted_path": "opposite path",
    "confirmation": "what confirms opposite",
    "invalidation": "what invalidates opposite"
  },
  "confidence_effect": "raise|cap|block"
}
```

规则：

- 每轮只保留一条主链和一条反链。
- 每条链最多三层解释。
- 只能使用 fresh facts、active events、静态 lesson rules。

## ExecutionRiskReview

```json
{
  "entry_orderable": true,
  "stop_valid": true,
  "risk_reward_ok": true,
  "manual_execution_risk": [],
  "event_hold_risk": [],
  "hard_blocks": [],
  "soft_downgrades": []
}
```

## DecisionPlan

模型最终输出的严格 JSON。

```json
{
  "instrument": "ETH-USDT-SWAP",
  "main_action": "trigger long",
  "horizon": "6h",
  "reference_price": 3500.0,
  "entry_trigger": 3510.0,
  "stop_price": 3435.0,
  "target_1": 3580.0,
  "target_2": 3660.0,
  "probability": 0.58,
  "position_size_class": "light",
  "max_leverage": 2,
  "risk_pct": 0.25,
  "expires_in_seconds": 900,
  "why_not_opposite": "...",
  "invalidation": "...",
  "unavailable_data": [],
  "manual_execution_required": true,
  "notes": "手动操作必需，系统没有下单。"
}
```

规则：

- `main_action` 必须是枚举。
- `manual_execution_required` 必须为 `true`。
- 新开仓、触发、反手必须有 stop。
- `probability` 不得超过 `EvidenceSnapshot.confidence_cap`。
- `expires_in_seconds` 必须大于 0，且不超过配置上限。

## RiskVerdict

```json
{
  "allowed": false,
  "status": "blocked|allowed|allowed_with_warnings",
  "reasons": [],
  "warnings": [],
  "confidence_cap_applied": 0.58
}
```

## JournalPayload

```json
{
  "request": {},
  "session_context": {},
  "skill_context": {},
  "evidence_snapshot": {},
  "root_cause_review": {},
  "execution_risk_review": {},
  "raw_model_output": "...",
  "decision_plan": {},
  "risk_verdict": {},
  "notification_result": {}
}
```

禁止写入：

- 完整 API key
- 完整 Bark key
- trade / withdraw credential
