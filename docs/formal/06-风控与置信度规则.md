# 风控与置信度规则

## 原则

风控规则必须由代码执行，不能只依赖 prompt。

模型可以解释原因，不能决定是否绕过 hard block。

## Hard Block

以下情况必须 block：

- `auto_order_enabled != false`
- `manual_execution_required != true`
- 存在 forbidden trade / withdraw key 环境变量
- symbol 不在 allowed symbols
- `main_action` 不在枚举中
- JSON 非法或字段缺失
- 新开仓、触发、反手没有 stop
- `expires_in_seconds <= 0`
- plan 已过期
- `max_leverage` 超过配置上限
- `risk_pct` 超过配置上限
- 核心市场事实缺失且是新开仓或触发动作
- journal 写入失败

## Soft Downgrade

以下情况可以不 block，但必须降级或降低置信度：

- liquidation heatmap 缺失
- precise CVD 缺失
- long/short ratio 只有 search-derived
- ETF flow 只有 preliminary 或 stale
- web fallback 可用但非 exchange-native
- active event window 内事件状态不完整
- order book 可用但深度不足

## Confidence Cap

代码必须计算并执行 confidence cap。

建议规则：

| 条件 | 上限 |
|---|---:|
| last / mark / index 缺失 | 0.59 |
| funding / OI 缺失且是杠杆方向判断 | 0.59 |
| active event 状态不明 | 0.55 |
| order book 缺失且是市场/高杠杆入场 | 0.55 |
| 只有 web-derived 衍生品数据 | 0.58 |
| 反向链同样强且无确认信号 | 0.56 |

如果模型输出 probability 高于 cap：

- v1 默认 block。
- 后续可配置为 clamp，但必须在 journal 和通知中标记。

## Action 风控

### open long / open short

必须满足：

- 核心价格完整。
- stop 存在。
- T1/T2 存在。
- RR 合理。
- 计划未过期。

### trigger long / trigger short

必须满足：

- trigger price 明确。
- stop 明确。
- invalidation 明确。
- trigger type 可执行或明确标记 recheck-only。

### hold long / hold short

必须满足：

- 已知用户持仓方向。
- 有失效条件。
- 有下一次复核时间。

### close long / close short

必须满足：

- 已知用户持仓方向。
- 说明为什么不是继续持有。

### flip long to short / flip short to long

v1 默认 hard block 或必须人工二次确认。

原因：

- 反手风险大。
- 容易被模型过度自信触发。

## 数据新鲜度

建议 freshness：

- price / mark / index：1-2 分钟内。
- funding / OI：5-15 分钟内。
- candles：必须对齐当前周期。
- liquidation / CVD / long-short：15-30 分钟内。
- breaking news：必须有发布时间。

超过 freshness：

- 如果是核心事实 -> block 或 cap。
- 如果是辅助事实 -> soft downgrade。

## 风控输出

```json
{
  "allowed": false,
  "status": "blocked",
  "reasons": ["missing mark price"],
  "warnings": ["liquidation heatmap unavailable"],
  "confidence_cap_applied": 0.58
}
```

## 通知措辞

如果被 block，通知必须写：

```text
该计划已被系统风控拦截，不应直接执行。
```

如果 allowed with warnings，通知必须写：

```text
该计划存在数据缺口或降级条件，请手动复核后再操作。
```
